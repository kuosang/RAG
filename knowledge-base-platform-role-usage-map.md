# 角色使用地图（内部 Agent 调试闭环版）

## 0. 文档定位
用于把“角色—页面—任务—交付物”连成可落地的使用地图，聚焦公司内部 Agent 的调试与闭环优化。

## 3. 角色旅程地图（内部 Agent 调试闭环）

### 3.1 主链路
1. **平台管理员**：新建知识空间与权限规则 → 输出“可用空间与边界”。
2. **知识管理员/运营**：配置模板与导入计划 → 输出“可生产知识的规范”。
3. **编辑/录入人员**：录入/导入知识 → 输出“草稿/待审核”。
4. **审核人员/业务负责人**：审核发布 → 输出“可引用知识”。
5. **AI 产品经理/Agent 配置**：在 `Query 测试` 里调整 MQA 阈值与策略 → 输出“可控的引用边界”。
6. **工作流编排/应用开发**：绑定节点调用 → 输出“上线可用的 Agent 工作流”。
7. **内部业务用户**：使用知识/Agent，提交反馈 → 输出“反馈与纠错数据”。
8. **知识管理员/运营**：创建维护任务 → 输出“更新后的知识版本”。

### 3.2 关键交接点
- **审核通过 → 调试**：审核发布后，AI 产品经理可开始调试阈值与策略。
- **负反馈 → 维护**：负反馈或低命中触发维护任务，回到知识编辑环节。
- **策略发布 → 监控**：策略上线后进入调用监控与质量分析。

---

## 4. 权限与边界原则（面向内部 Agent）
- 角色权限以 **空间级为主**，敏感空间需额外审批。
- 只有 **审核发布态知识** 可被 Agent 引用。
- MQA 阈值用于控制 **可引用边界**，阈值越高越偏向标准答案。
- 调试日志与操作记录需保留 **审计追溯**。

---

## 5. Demo 使用引导（快速展示）
- `Demo 控制台`：角色切换、阈值调整、关键弹窗演示。
- `Query 测试`：输入问题、调节阈值、观察可引用数量变化。
- `Agent 绑定`：展示策略与空间绑定。
- `维护任务`：从反馈回流到维护闭环。
