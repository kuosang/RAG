主流程：从平台配置到对外使用

这条线对应产品从“搭平台”到“产知识”再到“被消费 / 被接入”的完整正向链路。
主流程

1. Demo 控制台

总入口 / 讲解编排 / 场景切换
打开页面
  • 总览看板：统一说明六端范围、价值和推荐演示顺序。
  • 角色切换：按平台管理员、知识运营、算法调优、业务使用方推荐起点。
  • 页面地图 / 演示操作台：负责把演示流量分发到各功能页面。

2. 管理后台

平台治理 / 规则底座 / 系统收口
打开页面
  • 组织与权限：决定谁能建空间、调策略、发版本。
  • 空间 / 模板 / 数据源：决定知识如何被结构化地生产出来。
  • 模型与检索策略 / 审计：定义整个平台默认运行规则。

3. 知识运营工作台

知识生产 / 审核发布 / 生命周期维护
打开页面
  • 我的知识列表 / 新建知识 / 导入中心：负责内容录入和结构化补全。
  • 审核中心 / 版本与发布：负责把内容从草稿推进到可用版本。
  • 维护中心 / 反馈处理 / 质量分析：负责接住后续所有问题回流。

验证与消费：从可发布知识到可用答案

这条线负责把生产出来的知识先验证，再交给业务使用端和 Agent 接入端消费。
生产发布线

4. 检索调试台

检索验证 / 质量诊断 / 调优判断
打开页面
  • Query 测试 / 召回结果 / 重排分析：验证知识是否能被正确找到。
  • 引用链路 / 失败分析：定位为什么答错、没答到、引用不完整。
  • 评测集 / 评测结果：把一次次调优沉淀成持续验证机制。

5. 业务使用端

知识消费 / 搜索体验 / 人工反馈
打开页面
  • 首页搜索 / 分类导航 / 搜索结果：承接业务同学的直接找答案需求。
  • 知识详情 / FAQ 专题：承接阅读、引用和理解动作。
  • 收藏 / 最近浏览 / 反馈纠错：把消费行为重新变成优化输入。

6. Agent 集成端

智能接入 / 调用编排 / 运行监控
打开页面
  • Agent 列表 / 知识绑定：决定哪个 Agent 用哪些知识空间。
  • 检索策略 / Prompt / Workflow：决定知识如何被注入回答。
  • 调用监控 / API Key：承接线上调用和监控回流。

关键引导线说明

不是把页面并排放出来就结束，而是要明确每条连接线在产品上的含义。
正向交付线平台建设 → 知识上线 → 对外服务
Demo 控制台 → 管理后台
先定角色和演示场景，再进入平台治理能力说明底层规则从哪里来。
管理后台 → 知识运营工作台
治理端先建好空间、模板、词库、权限和数据源，运营端才能开始生产知识。
知识运营工作台 → 检索调试台
知识发布前后都需要进入调试台验证召回、重排、引用和失败问题。
检索调试台 → 业务使用端 / Agent 集成端
验证通过的知识，一部分给人工搜索消费，一部分给 Agent 自动接入使用。
回流优化线消费反馈 / 调用监控 / 调优回写
业务使用端 → 知识运营工作台
业务反馈、纠错、收藏热度和最近浏览会沉淀成内容优化任务。
Agent 集成端 → 检索调试台
线上调用日志、低分命中和零结果先回调试台分析根因。
检索调试台 → 管理后台
如果问题来自默认阈值、版本过滤、空间策略，就回到治理端统一收口。
检索调试台 → 知识运营工作台
如果问题来自别名不足、正文不全、引用字段缺失，就回到运营端补知识。

汇报时可直接照着讲的逻辑顺序

这部分把“平面图怎么讲”也固化出来，确保表达清晰。
推荐讲解脚本适合会议现场
第一步：总入口定场景
先用 Demo 控制台解释“这不是六个孤立页面，而是一套完整工作流”。
第二步：治理端定规则
说明平台管理员如何把权限、空间、模板、数据源和检索策略统一配置好。
第三步：运营端产知识
说明知识如何被创建、导入、审核、发布、维护并沉淀质量分析。
第四步：调试台保质量
说明每次发布后都要验证 Query、召回、重排、失败原因和评测结果。
第五步：双出口落价值
一条线给业务同学直接搜索消费，一条线给 Agent 自动接入调用。
第六步：反馈再回流
说明所有问题最终都能回到治理端或运营端形成持续优化闭环。
