# PRD 总文档（按模块拆解）

**产品名称**：面向 Agent 工作流的企业级知识中台

**版本**：V1.0（MVP 版本）

**状态**：草案 / 可评审

**编写日期**：2026-03-18

**适用范围**：管理后台、知识运营工作台、检索调试台、业务使用端、Agent 集成端

---

# 1. 背景与问题

当前企业知识分散在文档、IM、Wiki、工单、产品后台等多个系统中，存在以下痛点：
1. 口径不一致、版本混乱，影响业务决策与客户应答
2. 知识难以沉淀与复用，新人培训成本高
3. Agent / AI 工作流需要可信知识输入，但现有知识缺乏治理与结构化
4. 搜索效果不可控，召回不可解释，难以持续优化

本 PRD 目标是构建一个**面向 Agent 工作流的企业级知识中台**，打通“知识生产—治理—服务—优化”闭环。

---

# 2. 目标与非目标

## 2.1 目标（业务 + AI + 管理）
- 统一沉淀企业核心知识资产（产品、规则、SOP、FAQ、案例等）
- 建立审核、版本、责任人、权限、生命周期管理能力
- 为 Agent 提供结构化、可检索、可引用的知识服务
- 优先满足公司内部 Agent 调试与闭环优化需求
- 建立知识运营与效果评估机制（命中率、引用率、反馈闭环）

## 2.2 非目标（一期不做）
- 自动知识图谱构建
- 多模态深度内容理解（视频/语音）
- 全量外部系统连接器市场
- 完整多租户 SaaS 商业化能力

# 5. 模块级需求（按模块拆解）

以下 10 大模块是 PRD 的核心结构。

---

## 模块 01：知识空间管理

### 目标
建立知识的顶层组织结构，支撑业务域隔离与权限边界。

### 关键功能
- 新建/编辑/归档知识空间
- 多产品知识库切换/迁移（跨空间迁移）
- 目录树管理
- 空间负责人配置
- 空间权限配置（公开/受限）
- 空间绑定 Agent 状态查看

### 页面
- 知识空间管理页
- 空间详情抽屉 / 弹窗

### 业务规则
- 每个空间必须有责任人
- 归档空间不可被 Agent 调用
- 空间权限是文档权限的默认继承规则

### 异常与提示
- 空间名称重复提示
- 无权限查看空间内容提示

---

## 模块 02：知识录入与采集

### 目标
降低录入门槛，提高知识沉淀效率。

### 关键功能
- 模板化录入（FAQ / 产品说明 / SOP / 规则 / 案例）
- 富文本编辑
- 附件上传
- 批量导入（文件）
- 批量导出（按空间/条件）
- 导入预校验

### 页面
- 新建知识页
- 导入中心页（P1）

### 业务规则
- 未选择模板不可录入
- 必填字段缺失禁止提交审核
- 支持草稿自动保存

### 异常与提示
- 文件解析失败提示
- 字段缺失高亮提示

---

## 模块 03：知识编辑与结构化建模

### 目标
保证知识既可读又可检索，可被 Agent 精准召回。

### 关键功能
- 结构化字段补充
- 标签 / 分类维护
- 片段预览与重切片
- 关联知识

### 页面
- 知识详情编辑页

### 业务规则
- 发布态内容只能通过“新版本”修改
- 结构字段必须符合模板约束

---

## 模块 04：审核发布与版本管理

### 目标
确保发布内容可信、可追溯。

### 关键功能
- 草稿 / 待审 / 已发布 / 已下线 状态流转
- 审核通过 / 退回 / 转交
- 版本对比与回滚
- 生效/失效时间设置

### 页面
- 审核详情页
- 版本对比页（P1）

### 业务规则
- 草稿与审核态内容不可进入 Agent 可用池
- 审核通过即生成新版本号

---

## 模块 05：检索与召回中心

### 目标
保证“找得到、找得准”。

### 关键功能
- 全文检索
- 语义检索
- 混合召回
- 结果重排
- 高亮与片段预览
- MQA 阈值匹配与分数展示

### 页面
- 搜索结果页
- 召回结果页（调试）

### 业务规则
- 权限过滤必须在召回后执行
- 过期/下线知识不应返回给业务端
- 阈值用于控制 Agent 引用边界（阈值高则更偏向标准答案）

---

## 模块 06：Agent 集成中心

### 目标

### 关键功能

### 业务规则
- Agent 只能访问被绑定空间

# 6. 核心业务流程

## 6.1 知识生产链路
新建知识 → 补全字段 → 保存草稿 → 提交审核 → 审核通过 → 发布 → 进入检索池

## 6.2 Agent 接入链路
配置 Agent 绑定空间 → 配置策略 → Query 测试 → 召回验证 → Workflow 节点配置 → 上线

## 6.3 反馈与维护链路
业务端反馈 → 低质量识别 → 维护任务派发 → 修订 → 重新审核发布

# 9. 埋点与指标体系

## 9.1 核心指标
- 检索命中率 = 有结果查询数 / 总查询数
- 零结果率 = 无结果查询数 / 总查询数
- 引用率 = 有引用回答数 / Agent 总回答数
- 负反馈率 = 点踩数 / 总反馈数

## 9.2 必埋事件
- query_submitted
- query_result_viewed
- doc_opened
- doc_feedback_submitted
- agent_binding_updated
- retrieval_policy_updated
- review_task_completed

# 10. 非功能需求

- **性能**：普通搜索 < 2s，Agent 检索 < 4s

- **可观测**：检索日志、召回日志、引用日志

# 11. 版本规划与维护策略

## 11.1 一期（上线版本）
- 核心闭环：录入—审核—检索—Agent 调试—引用—反馈
- 多产品知识库管理：新增 / 删除 / 切换 / 迁移
- 批量导入与导出（含导出模板）
- MQA 阈值匹配（问题-答案匹配分数展示、阈值可调）
- 操作记录与审计可追溯
- 维护任务池与责任人机制

## 11.2 二期（增强版本）
- 外部系统连接器与自动同步
- 复杂模板与结构化字段扩展
- 高级检索评测集与 A/B 对比
- 更细颗粒度权限（字段级预留）

## 11.3 维护与更新机制
- 周期性触发过期 / 低命中 / 负反馈维护任务
- 重要知识变更走审核与版本回滚
- 关键指标驱动更新：命中率、引用率、负反馈率

# 12. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 知识无人维护 | 内容过期 | 责任人+维护任务+提醒机制 |
| Agent 调用效果差 | 业务不信任 | 调试台 + 引用链路可解释 |
| 权限混乱 | 数据泄露 | RBAC + 审计 + 过滤校验 |
| 项目过大 | 延期 | 严格按 MVP 范围推进 |
